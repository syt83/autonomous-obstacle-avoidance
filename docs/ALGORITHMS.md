## 1. Pure Pursuit (목표 추적)

### 📖 개요

Pure Pursuit은 경로 추종을 위한 기하학적 알고리즘으로, Look-ahead Point를 이용하여 차량의 조향각을 계산합니다.

### 🔬 동작 원리

1. 현재 차량 위치에서 전방 **Look-ahead Distance (ld)** 만큼 떨어진 경로상의 점을 찾습니다.
2. 차량에서 해당 점까지의 곡률 반경을 계산합니다.
3. 계산된 곡률을 이용하여 조향각 δ를 결정합니다.

### 📐 수식

조향각 계산:

```
δ = atan(2 * L * sin(α) / ld)
```

여기서:
- `δ`: 조향각 (steering angle)
- `L`: 휠베이스 (wheelbase)
- `α`: 차량 방향과 Look-ahead Point 사이의 각도
- `ld`: Look-ahead Distance

### ⚙️ 파라미터

| 파라미터 | 값 | 설명 |
|----------|-----|------|
| Look-ahead Distance (ld) | 튜닝 필요 | 너무 작으면 진동, 너무 크면 반응 느림 |
| 휠베이스 (L) | 0.025 m | 차량의 앞뒤 바퀴 간 거리 |

### 🎯 특징

**장점:**
- 구현이 간단하고 직관적
- 계산 비용이 낮음
- 부드러운 경로 추종

**단점:**
- Look-ahead Distance 튜닝이 중요
- 급격한 곡선에서는 성능 저하 가능

## 2. VFH - Vector Field Histogram (장애물 회피)

### 📖 개요

VFH는 LIDAR 센서 데이터를 이용하여 실시간으로 장애물을 감지하고 안전한 회피 방향을 선택하는 알고리즘입니다.

### 🔬 동작 원리

1. **LIDAR 데이터 수집**: 360° 범위에서 장애물까지의 거리 측정
2. **히스토그램 생성**: 각도별로 장애물 밀도를 계산하여 히스토그램 생성
3. **회피 방향 선택**: 장애물 밀도가 낮은 방향을 안전한 경로로 선택
4. **조향각 보정**: Pure Pursuit 출력에 회피 방향을 반영

### 📐 수식

장애물 밀도 계산:

```
H(θ) = Σ (m_i * (1 - d_i / d_max))
```

여기서:
- `H(θ)`: 각도 θ에서의 장애물 밀도
- `m_i`: 장애물의 중요도 (크기, 속도 등)
- `d_i`: 장애물까지의 거리
- `d_max`: LIDAR 최대 감지 거리

### ⚙️ 파라미터

| 파라미터 | 값 | 설명 |
|----------|-----|------|
| VFH 셀 크기 | 5° | 히스토그램의 각도 해상도 |
| LIDAR 감지 거리 | 1~2 m | 센서의 유효 감지 범위 |
| 안전거리 (d_safe) | 실측값 | 장애물과의 최소 안전 거리 |

### 🔍 히스토그램 예시

```
각도 범위:     0°   45°   90°  135°  180°  225°  270°  315°
밀도:         [0.8, 0.2, 0.1, 0.0, 0.0, 0.3, 0.5, 0.9]
                      ↑    ↑    ↑
                   안전한 방향 선택
```

### 🎯 특징

**장점:**
- 실시간 장애물 감지 및 회피
- 동적 환경에 효과적
- LIDAR 데이터 직접 활용

**단점:**
- 파라미터 튜닝 필요
- 좁은 공간에서는 제한적

## 3. LPF - Low Pass Filter (출력 안정화)

### 📖 개요

VFH의 출력은 장애물 감지 상황에 따라 급격하게 변할 수 있습니다. LPF는 이러한 급격한 변화를 완화하여 부드러운 조향을 가능하게 합니다.

### 🔬 동작 원리

1차 Low Pass Filter를 사용하여 VFH 출력을 필터링합니다.

### 📐 수식

```
y[k] = α * x[k] + (1 - α) * y[k-1]
```

여기서:
- `y[k]`: 현재 시점의 필터링된 출력
- `x[k]`: 현재 시점의 VFH 원본 출력
- `y[k-1]`: 이전 시점의 필터링된 출력
- `α`: 필터 계수 (0 < α ≤ 1)

### ⚙️ 파라미터

| 파라미터 | 값 | 설명 |
|----------|-----|------|
| 필터 계수 (α) | 0~1 | α=1: 필터 없음, α→0: 강한 필터링 |

### 📊 필터 효과

```
원본 신호:    ╱╲  ╱╲  ╱╲
              ╱  ╲╱  ╲╱  ╲

필터 적용:    ╱──╲  ╱──╲
            ╱      ╲╱    ╲
          (부드러운 곡선)
```

### 🎯 특징

**장점:**
- 조향 출력의 진동 제거
- 차량의 안정성 향상
- 구현이 매우 간단

**단점:**
- α 값 튜닝 필요
- 과도한 필터링 시 응답 지연

## 4. Differential Drive 변환

### 📖 개요

Pure Pursuit과 VFH로부터 계산된 조향각 δ를 Differential Drive 차량의 좌우 바퀴 속도 (VL, VR)로 변환합니다.

### 🔬 동작 원리

1. 조향각 δ를 각속도 ω로 변환
2. 전진 속도 Vx와 각속도 ω를 이용하여 좌우 바퀴 속도 계산

### 📐 수식

**Step 1: 조향각 → 각속도**

```
ω = (Vx * tan(δ)) / L
```

**Step 2: 각속도 → 좌우 바퀴 속도**

```
VL = Vx - (ω * W / 2)
VR = Vx + (ω * W / 2)
```

여기서:
- `ω`: 차량의 각속도 (angular velocity)
- `Vx`: 전진 속도 (0.5 m/s 고정)
- `L`: 휠베이스 (0.025 m)
- `W`: 트레드 (좌우 바퀴 간 거리)
- `VL`: 왼쪽 바퀴 속도
- `VR`: 오른쪽 바퀴 속도

### 🎯 예시

조향각 δ = 15° (오른쪽 회전):

```
Vx = 0.5 m/s
L = 0.025 m
W = 0.15 m (예시)

ω = (0.5 * tan(15°)) / 0.025 = 5.36 rad/s

VL = 0.5 - (5.36 * 0.15 / 2) = 0.098 m/s  (느림)
VR = 0.5 + (5.36 * 0.15 / 2) = 0.902 m/s  (빠름)

→ 오른쪽으로 회전
```

### ⚙️ 파라미터

| 파라미터 | 값 | 설명 |
|----------|-----|------|
| 전진 속도 (Vx) | 0.5 m/s | 고정 값 |
| 휠베이스 (L) | 0.025 m | 앞뒤 바퀴 간 거리 |
| 트레드 (W) | 실측값 | 좌우 바퀴 간 거리 |

### 🎯 특징

**장점:**
- Differential Drive의 정확한 운동학 모델
- 정밀한 회전 제어 가능

**단점:**
- 차량 파라미터 정확도에 의존

## 🔄 통합 시스템 흐름

```
1. LIDAR 센서 데이터 수집
   ↓
2. VFH로 장애물 밀도 히스토그램 생성
   ↓
3. 안전한 회피 방향 선택
   ↓
4. LPF로 출력 안정화
   ↓
5. Pure Pursuit으로 조향각 δ 계산
   ↓
6. Differential Drive 변환 (δ → VL, VR)
   ↓
7. 차량 구동
```
